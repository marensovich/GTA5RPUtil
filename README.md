# GTA5RP Util — Проверка судимостей (с GUI и OCR)

Программа с графическим интерфейсом (Qt/PySide6, тёмная тема, боковая
навигация) для автоматизации поиска паспортов в игровой панели и
проверки судимостей персонажей через распознавание текста (OCR,
Tesseract) — без ручных нажатий стрелок и без ручного чтения таблицы
результатов.

## GUI на Qt (PySide6)

Интерфейс сделан на **PySide6** (официальные Python-биндинги Qt) —
сознательный выбор в пользу качества визуала: настоящие полупрозрачные
тени карточек, градиенты и hover/focus-состояния через QSS (Qt-аналог
CSS) вместо ручной отрисовки через PIL на Canvas. Обратная сторона —
размер exe заметно вырос (Qt сам по себе тяжёлая библиотека), см. раздел
"Из чего складывается размер exe" ниже.

Поток автоматизации общается с интерфейсом через `QThread` + сигналы/
слоты Qt (`AutomationWorker` в `gui.py`) — это потокобезопасно "из
коробки".

## Как это работает

1. Программа находит окно процесса (по умолчанию `GTA5.exe`, выбирается
   из выпадающего списка реально открытых окон), разворачивает его и
   выводит на передний план.
2. Ищет на экране поле "Номер паспорта" и кнопку "Поиск" по картинкам-
   шаблонам (`templates/`).
3. Берёт список записей — либо из входного файла, либо из текста,
   вставленного прямо в программу (вкладка "Проверка судимостей",
   переключатель "Из файла" / "Вставленный текст") — построчно вытаскивая
   значение в круглых скобках, например из строки `Oper Aether (400560) 7`
   — `400560`. Текст, вставляемый в поле, всегда вставляется как обычный
   белый текст без форматирования (никаких унаследованных цветов/шрифтов
   из буфера обмена).
4. Для каждой записи: активирует окно, очищает поле, вводит номер, жмёт
   "Поиск".
5. Находит на экране таблицу результатов (по шапке колонки "МЕСТО
   ОТБЫВАНИЯ НАКАЗАНИЯ"), распознаёт текст через OCR и ищет в нём фразы,
   относящиеся к выбранному режиму сортировки (см. ниже). Если нужных
   фраз не видно — прокручивает список вниз и проверяет снова, пока не
   найдёт совпадение или пока список не закончится.
6. Совпавшие записи сразу появляются в поле "Результаты" (цветом —
   какая категория найдена) и в живом прогресс-баре — **в файл ничего не
   сохраняется**, результаты живут только в интерфейсе программы, пока
   она открыта.
7. По завершении — сводка (обработано/найдено по категориям, статистика
   времени) и системное уведомление (см. "Уведомление о завершении"
   ниже).

## Режимы сортировки и категории результатов

- **Административные (АК)** — ищет записи, где "Место отбывания
  наказания" начинается с `КПЗ` (любой участок — `КПЗ LSPD`, `КПЗ LSSD`
  и т.д.).
- **Уголовные (УК)** — ищет `следственный изолятор` / `федеральная
  тюрьма`.
- **Оба вида (раздельно)** — проверяет оба списка ключевых слов; в
  результатах каждая запись помечается своей категорией: `АК`, `УК`, или
  `УК + АК`, если у персонажа есть и то, и другое.

Ключевые слова для обеих категорий можно поменять на вкладке
"Настройки".

## Запуск

### Вариант А — готовый .exe (рекомендуется для обычного использования)

Если вам уже собрали exe (см. раздел "Сборка в .exe" ниже) — просто
запустите его. Отдельно устанавливать Tesseract-OCR **не нужно** — он
встроен внутрь exe. Положите `input.txt` рядом с exe (или укажите путь к
нему в программе, либо вообще вставьте список записей текстом на вкладке
"Проверка судимостей").

### Вариант Б — запуск из исходников (для разработки)

```powershell
.venv\Scripts\python.exe -m pip install -r requirements.txt
.venv\Scripts\python.exe main.py
```

При запуске из исходников программа сама попробует найти установленный
отдельно Tesseract-OCR (по умолчанию `C:\Program Files\Tesseract-OCR\`) —
см. "Установка Tesseract-OCR" ниже — либо использует папку `tesseract_bin\`,
если она есть в проекте (та же копия, что упаковывается в exe).

## Сборка в .exe (один файл, с встроенным Tesseract-OCR)

Чтобы конечным пользователям не нужно было ставить Tesseract-OCR отдельно,
он упаковывается внутрь exe через PyInstaller.

1. Подготовьте папку `tesseract_bin\` в корне проекта — урезанная
   портативная копия Tesseract-OCR: только реально нужные для распознавания
   файлы (без документации, инструментов обучения, GUI-зависимостей вроде
   pango/cairo/ICU и без лишних языков — используется только `rus`):
   ```
   tesseract_bin/
   ├── tesseract.exe
   ├── *.dll                  (только DLL, от которых реально зависит tesseract.exe —
   │                            список проверен через разбор PE-заголовков, см. ниже)
   └── tessdata/
       └── rus.traineddata
   ```
   Проще всего — скопировать `tesseract.exe` и `tessdata\rus.traineddata` из
   уже установленного `C:\Program Files\Tesseract-OCR\` (поставьте галочку
   "Russian" при установке — см. ниже). Список реально нужных DLL был
   получен разбором PE-заголовков (`pip install pefile`, рекурсивный обход
   `IMAGE_DIRECTORY_ENTRY_IMPORT` от `tesseract.exe`), а не на глаз — так
   что просто копировать все DLL из полной поставки не нужно, среди них
   много незадействованных (ICU, pango/cairo и т.п.). Эта папка уже
   подготовлена и урезана в репозитории (~29 МБ вместо ~86 МБ у полной
   поставки Tesseract-OCR).

2. Запустите сборку:
   ```powershell
   .\build.ps1
   ```
   Скрипт сам установит PyInstaller в `.venv`, соберёт `main.py` в один
   exe-файл и добавит внутрь `templates\`, `tesseract_bin\` и `logo.jpg`.

3. Готовый файл: `dist\GTA5RPUtil.exe`. Его можно скопировать на другой
   компьютер с Windows — Tesseract-OCR и Python устанавливать не нужно.
   Понадобится только сам `input.txt` рядом с exe (или список записей,
   вставленный вручную в программе) и запущенная игра.

### Из чего складывается размер exe

Итог: **~88 МБ**.

- Урезанный Tesseract-OCR (`tesseract_bin\`) — ~29 МБ.
- PySide6 (QtCore, QtGui, QtWidgets) — основной вклад в рост. В
  `build.ps1` явно исключены модули Qt, которые не используются
  (QtQml, QtQuick, QtWebEngine*, QtMultimedia, QtNetwork, QtSql,
  QtPdf, QtCharts и др.) — без этого размер был бы заметно больше,
  PySide6 тянет за собой весь Qt по умолчанию.
- Python-рантайм, Pillow, numpy, pywin32 — остальное.
- **OpenCV сознательно не используется.** Раньше поиск полей/кнопок на
  экране (`pyautogui` с `confidence=...`) требовал `opencv-python`, а это
  ~85 МБ веса ради одной функции (`cv2.matchTemplate`). Вместо неё —
  `element_matcher.py`: та же нормализованная кросс-корреляция
  (TM_CCOEFF_NORMED), но реализованная на numpy (БПФ + интегральное
  изображение, обе части в float64 — на float32 давало численную
  нестабильность на больших мультимониторных кадрах). Проверено на
  синтетических тестах — совпадает с OpenCV по позиции и confidence с
  точностью до третьего знака. Из-за этого поиск шапки таблицы
  результатов на весь экран стал медленнее (десятые доли секунды вместо
  ~0.05 с на запись), но экономия в весе exe того стоит.

## Установка Tesseract-OCR (только для разработки из исходников)

Нужно, только если вы запускаете `python main.py` напрямую и в проекте
ещё нет папки `tesseract_bin\`. Собранный `.exe` в этом не нуждается.

1. Скачайте установщик для Windows: https://github.com/UB-Mannheim/tesseract/wiki
2. Во время установки **обязательно поставьте галочку "Russian"** в списке
   дополнительных языковых пакетов (Additional language data) — без него
   OCR не будет распознавать кириллицу.
3. По умолчанию программа ставится в
   `C:\Program Files\Tesseract-OCR\tesseract.exe`. Если у вас другой путь —
   укажите его на вкладке "Настройки" → "Путь к tesseract.exe".

## Интерфейс

Слева — боковая навигация с четырьмя разделами:

- **Проверка судимостей** — всё в одном месте: выбор режима сортировки
  (АК / УК / оба), настройки игры (процесс, активация окна, захват
  мониторов, оверлей поверх игры, горячая клавиша остановки), источник
  входных записей (файл или вставленный текст), кнопки Старт/Стоп,
  прогресс-бар и поле "Результаты", куда в реальном времени добавляются
  найденные совпадения (цветом — категория: `АК` / `УК` / `УК + АК`).
  Никакой постоянной статистической панели нет — сводка (обработано,
  найдено по категориям, время на человека) появляется под прогресс-баром
  только по завершении прогона.
- **Фракция** — в разработке (заглушка "Скоро…").
- **Лог** — подробный текстовый журнал выполнения (что происходит на
  каждом шаге, ошибки, предупреждения о калибровке, статистика времени
  по завершении прогона).
- **Настройки** — все тонкие настройки (см. ниже) с пояснениями под
  каждым полем, прокручивается, если не помещается на экран.

### Горячая клавиша аварийной остановки

По умолчанию **F3** (поле "Горячая клавиша аварийной остановки" на
вкладке "Проверка судимостей") — останавливает обработку даже когда окно
программы не в фокусе (например, активна игра). Работает через пакет
`keyboard` (глобальный хук клавиатуры); если он не установлен, в логе
появится предупреждение и хоткей просто не сработает — кнопка "Стоп" в
интерфейсе продолжит работать как обычно.

### Оверлей поверх игры

Если включён флажок "Показывать оверлей с прогрессом поверх игры"
(вкладка "Проверка судимостей", включён по умолчанию) — во время прогона
поверх всех окон (в т.ч. игры) появляется небольшое полупрозрачное
окошко без рамки: прогресс `X/Y`, кто проверяется сейчас, сколько времени
прошло/осталось (оценка), среднее/минимальное/максимальное время на
человека и скорость (чел/мин). Окошко можно перетащить мышью на удобное
место; по умолчанию встаёт в правый верхний угол окна игры (если оно
найдено) либо экрана. Скрывается через несколько секунд после завершения
прогона.

### Уведомление о завершении

По завершении (или остановке/ошибке) прогона программа показывает
системное уведомление (всплывающее окно Windows) с краткой сводкой — это
удобно, если окно программы свёрнуто или перекрыто игрой. Уведомление
использует системный трей; иконка в трее включается только на время
показа сообщения и затем сама скрывается — постоянно в трее программа не
висит.

### Настройки (вкладка "Настройки")

Все параметры, которые раньше были константами в коде, редактируются в
интерфейсе и сохраняются в `gui_settings.json` рядом с программой.
Значения точности распознавания и всех пауз задаются ползунками (с живым
отображением текущего значения) вместо голых чисел — под каждым есть
короткое пояснение, что регулирует параметр и когда его стоит менять.
Значения по умолчанию (`settings.py`):

| Параметр | По умолчанию | Смысл |
|---|---|---|
| Точность распознавания (confidence) | 0.8 | Порог совпадения шаблона поля/кнопки/шапки на экране |
| Пауза между действиями | 0.1 с | Клики, ввод текста, активация окна |
| Пауза после активации окна | 0.1 с | Между переключением фокуса на игру и первым кликом |
| Пауза загрузки результатов | 0.3 с | Между нажатием "Поиск" и началом чтения таблицы |
| Пауза после прокрутки | 0.1 с | Список результатов |
| Backspace при очистке поля | 30 раз | Подстраховка на случай, если Ctrl+A не сработал |
| Макс. попыток прокрутки | 15 | Список результатов |
| Величина прокрутки | -400 | Список результатов |

Плюс: область таблицы результатов (смещения/размер — калибровка под ваш
экран, см. ниже), путь к `tesseract.exe` (переопределение автоопределения),
ключевые слова для административных/уголовных судимостей, показывать ли
оверлей поверх игры, сохранение отладочных скриншотов OCR в папку
`debug/`.

## Калибровка области распознавания (если OCR не находит текст)

1. На вкладке "Настройки" включите "Сохранять отладочные скриншоты и
   текст OCR в папку debug/".
2. Запустите обработку на 1-2 записях, затем откройте `debug/` — там будут
   картинка, которую видел OCR (включая отдельные картинки с отмеченной
   рамкой для поиска поля/кнопки/шапки), и распознанный текст. Сверьте,
   попадает ли в кадр колонка "Место отбывания наказания" целиком, и туда
   ли вообще попала рамка совпадения.
3. Если область смещена — подправьте смещения/размер области результатов
   на той же странице. Лог также показывает оценку совпадения (score) для
   поля/кнопки/шапки — если она заметно ниже confidence, шаблоны в
   `templates/` не соответствуют вашему экрану и их нужно переснять.

## Работа с несколькими мониторами

Флажок "Захват всех мониторов" (включён по умолчанию) — программа
захватывает весь виртуальный рабочий стол (все мониторы) и сама
пересчитывает координаты кликов и прокрутки в реальные координаты
Windows.

## Игровой режим

Рекомендуется **оконный без рамки** (Windowed Borderless), не полноэкранный
эксклюзивный режим — в эксклюзивном фуллскрине скриншоты через Windows API
могут не захватываться корректно.

## Структура проекта

```
GTA5RPUtil/
├── main.py                          # точка входа — запускает GUI
├── gui.py                           # графический интерфейс (PySide6/Qt): главное окно + оверлей поверх игры
├── automation.py                    # логика: активация окна, OCR, классификация судимостей, статистика времени
├── element_matcher.py               # поиск шаблона на экране без OpenCV (numpy, FFT + интегральное изображение)
├── settings.py                      # настройки (Settings), режимы сортировки
├── ocr_setup.py                     # определение путей ресурсов и Tesseract (dev/exe)
├── updater.py                       # версия приложения + проверка обновлений через GitHub Releases API
├── build.ps1                        # сборка в один .exe (PyInstaller)
├── requirements.txt                 # Python-зависимости
├── input.txt                        # список записей для обработки (пример)
├── gui_settings.json                # настройки, сохранённые из интерфейса (создаётся автоматически)
├── logo.jpg                         # логотип: иконка окна + маленькая иконка в шапке
├── tesseract_bin/                   # встраиваемая портативная копия Tesseract-OCR
├── debug/                           # (опционально) скриншоты + текст OCR для калибровки
└── templates/
    ├── field_template.png           # эталон поля "Номер паспорта"
    ├── button_template.png          # эталон кнопки "Поиск"
    └── results_header_template.png  # эталон шапки колонки "Место отбывания наказания"
```

Результатов в файловом виде больше нет — вывод существует только в
интерфейсе программы (вкладка "Проверка судимостей") и в системном
уведомлении по завершении.

## Статистика времени

По завершении (или остановке) прогона в лог выводится сводка: сколько
всего затрачено времени, среднее/минимальное/максимальное время на
одного человека и скорость проверки (человек в минуту и в секунду). Та
же статистика в реальном времени видна в оверлее поверх игры и в сводке
под прогресс-баром на вкладке "Проверка судимостей".

## Проверка обновлений

При запуске программа в фоновом потоке (не блокируя интерфейс) один раз
стучится в GitHub Releases API репозитория и сравнивает версию с зашитой
в `updater.py` (`APP_VERSION`). Если найдена более новая версия — вверху
появляется закрываемая плашка со ссылкой на скачивание. Автообновления
нет (exe не может сам себя подменить на Windows без отдельного
лаунчера) — только уведомление, обновляться нужно вручную. Нет
интернета / нет ни одного релиза / GitHub недоступен — проверка просто
молча ничего не показывает, никаких ошибок пользователю.

Чтобы уведомление работало, при выпуске новой версии:
1. Поднимите `APP_VERSION` в `updater.py`.
2. Создайте на GitHub релиз с тегом вида `v1.1.0` (`git tag` + `gh release create`,
   либо через веб-интерфейс) — API берёт именно "latest" релиз.
